Getting Started

Deploy your server, scan the QR code, start coding. The whole process takes about 5 minutes.

What you need

  • ✓ Continuum installed on your iPhone or iPad
  • ✓ A Mac or Linux machine to run the deploy script (your laptop is fine)
  • ✓ gum — a terminal UI tool used by the deploy script (brew install gum)
  • ✓ WireGuard tools — for key generation (brew install wireguard-tools qrencode)
  • ✓ For cloud deploys: the provider's CLI (aws, doctl, or hcloud)

Choose your setup path

Continuum works with any Linux server you can SSH into. Pick the path that fits your situation.

☁️ New cloud server (recommended)

Spin up a fresh VM on AWS Lightsail, EC2, DigitalOcean, or Hetzner. The script handles everything.

  1. Clone the server repo and run the deploy script: 
    git clone https://github.com/kalleeh/continuum-ios.git
cd continuum-ios/server
./deploy.sh create lightsail
    The script will ask you to pick a region, instance size, and number of devices. Replace lightsail with ec2, do, or hetzner for other providers.
  2. Wait ~3 minutes for the VM to boot and configure itself. The script will show progress.
  3. When it's done, a QR code appears in your terminal. This contains your WireGuard config and auth token.
  4. Open Continuum on your iPhone → tap Scan QR Code → point at the terminal. Done.

🖥️ Existing server (any Linux VPS)

Already have a server running? The script will SSH in and set everything up.

  1. Clone the repo and run: 
    git clone https://github.com/kalleeh/continuum-ios.git
cd continuum-ios/server
./deploy.sh create any
  2. The script will ask for your server's IP and SSH key. It installs WireGuard, ttyd, and the Continuum relay over SSH.
  3. Scan the QR code that appears when it finishes.

💻 Your own Mac or Linux machine

Run everything locally — great for trying it out or if your dev machine is always on.

  1. Clone the repo and run: 
    git clone https://github.com/kalleeh/continuum-ios.git
cd continuum-ios/server
./deploy.sh create local
  2. The script installs WireGuard and the relay services on your machine.
  3. Scan the QR code. Your phone connects to your machine over WireGuard.

After scanning the QR code

The app imports your WireGuard config and auth token automatically. The VPN connects, and you'll see the session list within a few seconds.

From there:

  • + Tap New Session to create a Claude Code, Kiro, Ollama, or terminal session
  • + Sessions run on your server inside tmux — they keep running even when you close the app
  • + Come back anytime and pick up exactly where you left off

Troubleshooting

The QR code won't scan
Make sure your terminal window is large enough for the QR code to render clearly. Try zooming in or increasing your terminal font size. You can also re-display the QR code with ./deploy.sh status.
VPN connects but no sessions appear
The relay service may still be starting. Wait 30 seconds and pull down to refresh. Check that the relay is running on your server: systemctl status continuum-relay.
"Connection failed" after scanning
Your phone needs to be on a network that can reach your server. If you're on the same local network as a create local setup, make sure your firewall allows UDP port 51820 (WireGuard).
I want to add another device
Run ./deploy.sh create again with --peers 2 (or however many devices you need). Each device gets its own QR code.
How do I update the server?
Run ./deploy.sh status to check versions. The relay binary auto-updates, or you can re-run the deploy script.
